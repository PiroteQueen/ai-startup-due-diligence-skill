# 中文体系化尽调输出模板

> Purpose: final Chinese-facing DD report for investors/founders. Use this after the internal evidence ledger, external research log, competitor landscape, module coverage, red-team checks, and decision rules are complete. Do not expose every internal module unless the user asks for the appendix.

## 0. 一句话主问题

> 这家公司/这笔交易到底要证明什么，才值得继续推进？

写法要求：
- 用一个中文问题串起全篇。
- 不要写成泛泛的“是否值得投资”。
- 问题必须同时包含业务本质、当前阶段、关键不确定性。

示例：
> 这家公司是否已经从“AI 演示能力”进入“真实工作流锁定”，并且当前估值仍然给投资人留下足够回报空间？

## 1. 一句话结论

- 当前建议：Proceed / Watch / Pass / Need more evidence
- 置信度：high / medium / low
- 结论必须说明“为什么现在是这个动作”。

机器可校验锚点（ASCII 冒号，勿改字段名）：

- Verdict: Proceed / Watch / Pass / Need more evidence
- This verdict reverses if:

示例：
> 当前建议为 Watch / Need more evidence：方向和团队值得继续跟踪，但交易条款、PMF 证据和竞品替代优势尚未达到投资承诺门槛。

## 2. 五道门判断

八模块 coverage 用于后台检查完整性；中文正式输出只保留五道门。

| 门 | 当前判断 | 已验证证据 | 关键未知 | 下一步动作 |
|---|---|---|---|---|
| 1. 交易真实性 | 真实 / 待核验 / 不成立 | term sheet、cap table、allocation、SPV/直投结构 | ... | ... |
| 2. 团队与组织能力 | 强 / 中 / 弱 / 待核验 | 创始人角色、核心团队、招聘、执行记录 | ... | ... |
| 3. 产品技术与 AI 战略 | 已产品化 / 原型 / 演示 / 叙事 | 产品工作流、AI 功能状态、采用度、benchmark、eval、架构、数据权利、模型依赖 | ... | ... |
| 4. 市场与部署证据 | 有 PMF / 有试点 / 有需求线索 / 未证实 | 客户、收入、留存、LOI、部署 | ... | ... |
| 5. 估值与回报空间 | 合理 / 偏贵 / 透支 / 无法判断 | 融资额、投前投后、退出可选性 | ... | ... |

## 3. 证据状态

把每个高重要性 claim 标成四类之一：

| Claim | 状态 | 来源 | 备注 |
|---|---|---|---|
| ... | 已验证 / 材料声称 / 推断 / 未知或矛盾 | provided / external / data room / interview | ... |

规则：
- “材料声称”不能写成“事实”。
- “外部有报道”不等于“交易条款已验证”。
- “demo 可看”不等于“PMF 已成立”。
- “客户 logo”不等于“付费部署”。

## 4. 投资判断主线

用 3–6 段中文说明：

1. 这个资产真正想成为的东西是什么；
2. 当前已经证明了什么；
3. 当前还没有证明什么；
4. 哪一层风险最大；
5. 估值是否已经提前透支成功；
6. 什么证据会改变结论。

另设“AI 产品与能力战略”小节，单独说明：

- 当前已上线、测试中、仅宣布和已下线的 AI 功能；
- AI 在公司战略中的角色；
- 自研、采购、开源与合作边界；
- 采用度、商业化和单位经济；
- 相对竞品的真实差异与平台依赖。

避免：
- 直接粘贴八模块表；
- 中英文术语混排；
- 模板堆叠；
- 只有风险没有下一步；
- 只有乐观叙事没有证伪条件。

## 5. 下一步资料室 / 访谈 / 外部核验

只列能改变判断的下一步，不列无限清单。

| 优先级 | 请求/动作 | 解决哪个问题 | 如果拿不到怎么办 |
|---|---|---|---|
| P0 | ... | ... | 结论降级 / 暂停 / Pass |
| P1 | ... | ... | ... |
| P2 | ... | ... | ... |

## 6. 最终输出句式

结尾必须给出一个可执行动作：

- 进入资料室，但不承诺投资；
- 安排创始人/客户/技术访谈；
- 等待下一阶段里程碑；
- 直接拒绝；
- 改为非 VC 路径或战略合作观察。

示例：
> 因此，当前不建议进入投资承诺。建议先要求资料室，重点核验交易条款、核心团队全职投入、产品/技术证据、客户部署与竞品替代优势。若 P0 资料无法提供，结论应从 Watch 降为 Pass。
